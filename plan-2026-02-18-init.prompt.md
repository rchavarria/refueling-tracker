## Plan: Car Fuel Consumption Tracker Web Application

Develop a full-stack TypeScript application with React frontend, Node.js/Express backend, and SQLite,
using Biome for linting, Prisma ORM with migrations and seed data, Tailwind CSS, Chart.js, Zod schemas
in `shared/` folder, Vitest for testing, and Docker Compose. Vehicles have unique and required `name`,
refuelings record validated total mileage, and distance traveled is calculated dynamically showing "N/A"
for first refueling or when two consecutive refuelings have the same mileage.

### Steps

Steps will be implemented in order, with clear commit messages for each step.
Each step will include code implementation, testing, and documentation as needed.

1. Configure monorepo with `frontend/` (React + Vite + TypeScript + Tailwind), `backend/` (Node +
   Express + TypeScript + Vitest), `shared/` (Zod schemas), including `biome.json` at root,
   `tsconfig.json` for each project, workspaces in root `package.json` with dev/build/test/db:reset
   scripts
2. Initialize backend with Prisma, create schema with `Vehicle` (id, name unique not null, brand, model,
   licensePlate, year) and `Refueling` (id, vehicleId FK, date, liters, totalPrice, mileage, station),
   run `prisma migrate dev --name init`
3. Create `prisma/seed.ts` with sample data using dates from January-February 2026: 2-3 vehicles with
   unique names, 5-7 refuelings per vehicle with incremental mileage (increments 300-800 km), realistic
   prices (1.50-1.80 €/L), configure `package.json` with `db:seed` and `db:reset` scripts (reset +
   migrate + seed)
4. Create Zod schemas in `shared/schemas/`: `vehicleSchema` with `name` string min(1) required,
   `refuelingSchema` with `mileage` number positive, configure `tsconfig.json` with paths `@shared/*`
   for shared imports
5. Implement REST API in `backend/src/`: controllers with Zod validation, middleware to validate
   `mileage` > last vehicle mileage (return 400 error), endpoints `/api/vehicles` (POST validates `name`
   uniqueness), `/api/refuelings`, `/api/vehicles/:id/refuelings` ordered by `date ASC`, `/api/health`
   healthcheck
6. Create `backend/src/services/statistics.service.ts` with `calculateConsumption()` function: receives
   ordered refuelings array, calculates km traveled between consecutive refuelings, returns L/100km and
   €/km rounded to 2 decimals, handles first refueling and equal mileage returning null
7. Implement Vitest tests in `backend/src/services/statistics.service.test.ts`: cases for first refueling
   (returns null), multiple refuelings (correct calculation), consecutive equal mileages (returns null),
   rounding to 2 decimals
8. Develop TypeScript frontend: React Router (routes `/`, `/vehicles`, `/vehicles/:id`, `/refuelings/new`),
   Tailwind components (VehicleForm, RefuelingForm, RefuelingList showing "N/A" when km traveled is null,
   table: date, liters, €, total km, km traveled, L/100km), Dashboard with Chart.js (consumption timeline,
   cost/km bars)
9. Create `docker-compose.yml` with services `frontend` (port 5173), `backend` (port 3000), volume
   `sqlite-data`, multi-stage `Dockerfile` for each service, `.env.example` with DATABASE_URL,
   VITE_API_URL, NODE_ENV, PORT

### Further Considerations

1. **Clear error messages**: Mileage validation should return specific message like "Mileage must be
   greater than the last recorded (X km)"
2. **Date formatting**: In frontend use Spanish format dd/mm/yyyy for display, ISO for API communication
3. **Docker persistence**: SQLite volume should mount at `/app/backend/prisma/dev.db` to persist data
   between restarts
4. **Hot reload in development**: Configure Docker Compose with bind mounts for development with automatic
   reload in frontend and backend

### Technology Stack

- **Frontend**: React, TypeScript, Vite, Tailwind CSS, Chart.js, React Router, Zod, Fetch API
- **Backend**: Node.js, Express, TypeScript, Prisma ORM, SQLite, Zod, Vitest
- **DevOps**: Docker Compose, Biome (linting), shared Zod schemas
- **Features**: Vehicle management (unique name), refueling tracking (validated mileage), consumption
  statistics (L/100km, €/km), visual charts
